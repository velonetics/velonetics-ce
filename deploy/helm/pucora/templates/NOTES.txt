1. Get the Pucora API Gateway URL:

{{- if .Values.ingress.enabled }}
{{- range $host := .Values.ingress.hosts }}
  {{- range .paths }}
  http{{ if $.Values.ingress.tls }}s{{ end }}://{{ $host.host }}{{ .path }}
  {{- end }}
{{- end }}
{{- else if contains "NodePort" .Values.service.type }}
  export NODE_PORT=$(kubectl get --namespace {{ .Release.Namespace }} -o jsonpath="{.spec.ports[0].nodePort}" services {{ include "pucora.fullname" . }})
  export NODE_IP=$(kubectl get nodes --namespace {{ .Release.Namespace }} -o jsonpath="{.items[0].status.addresses[0].address}")
  echo http://$NODE_IP:$NODE_PORT
{{- else if eq (include "pucora.isLoadBalancerService" .) "true" }}
  NOTE: It may take a few minutes for the LoadBalancer IP/hostname to be available.
  kubectl get --namespace {{ .Release.Namespace }} svc -w {{ include "pucora.fullname" . }}
{{- if .Values.service.nlb.enabled }}

  NLB mode is enabled. Check cloud-provider tags and metadata via service annotations.
{{- end }}
{{- else }}
  kubectl port-forward --namespace {{ .Release.Namespace }} svc/{{ include "pucora.fullname" . }} 8080:{{ .Values.service.port }}
  echo "Visit http://127.0.0.1:8080"
{{- end }}

2. Check the health endpoint:

  curl http://127.0.0.1:8080/__health

   Update config.pucoraJson in values.yaml or use --set-file config.pucoraJson=./pucora.json
{{- else if eq .Values.config.mode "configurator" }}
   Gateway loads config from Configurator API (config.configurator.configName={{ .Values.config.configurator.configName }}).
   Restart gateway pods after publishing changes in the UI.
{{- else if eq .Values.config.mode "secret" }}
   Update config.pucoraJson or config.existingSecret for Secret-backed configuration.
{{- else }}
   Configuration is expected to be baked into the container image.
